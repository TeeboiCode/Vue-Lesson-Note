Lesson 01

Introduction to Vue.js

What is Vue.js and why use it?

What is Vue.js?

Vue.js (pronounced "view") is a progressive JavaScript framework for building user interfaces. Unlike other monolithic frameworks, Vue is designed from the ground up to be incrementally adoptable.

Vue focuses on the view layer only, making it easy to integrate with other libraries or existing projects. It's also perfectly capable of powering sophisticated Single-Page Applications (SPAs) when combined with modern tooling and supporting libraries.

Why Choose Vue.js?

  • Easy to Learn — Simple syntax and clear documentation make it beginner-friendly
  • Reactive Data Binding — Automatically updates the DOM when data changes
  • Component-Based — Build reusable, encapsulated components
  • Lightweight — Small bundle size (~20KB min+gzip)
  • Versatile — Can be used for simple widgets or complex applications

Vue.js vs Other Frameworks

Framework Learning Curve Size Best For
Vue.js Easy ~20KB SPAs, Progressive Enhancement
React Moderate ~40KB Large-scale applications
Angular Steep ~60KB Enterprise applications

The Vue Ecosystem

Vue.js has a rich ecosystem of official and community tools:

  • Vue Router — Official routing library
  • Vuex / Pinia — State management solutions
  • Vue CLI — Standard tooling for Vue projects
  • Vite — Next-generation build tool
  • Nuxt.js — Framework for server-side rendering
💡
Good to know

Vue.js was created by Evan You in 2014 after working on AngularJS at Google. He wanted to extract the parts he liked about Angular and build something lightweight.

Your First Vue Application

Here's the simplest possible Vue app:

<div id="app">{{ message }}</div>

<script src="https://unpkg.com/vue@3"></script>
<script>
  const { createApp } = Vue;
  
  createApp({
    data() {
      return {
        message: 'Hello Vue!'
      }
    }
  }).mount('#app');
</script>

This code creates a Vue application that displays "Hello Vue!" on the page. The {{ message }} syntax is called interpolation — it displays the value of the message property.

Getting Started/Setup

Setting up your Vue development environment

Installation Methods

There are several ways to use Vue.js in your projects:

1. CDN (Content Delivery Network)

The quickest way to get started — perfect for learning and prototyping: 

<!-- Vue 3 -->
<script src="https://unpkg.com/vue@3"></script>

<!-- Or specific version -->
<script src="https://unpkg.com/vue@3.3.4/dist/vue.global.js"></script>

2. NPM (Node Package Manager)

For building large-scale applications with build tools:

# Install Vue
npm install vue@latest

# Or with Yarn
yarn add vue

3. Create Vue Project (Recommended)

The official scaffolding tool for Vue projects with Vite (fast build tool): 

# Create a new project
npm create vue@latest

# Follow the prompts:
# ✔ Project name: my-vue-app
# ✔ Add TypeScript? No
# ✔ Add JSX Support? No
# ✔ Add Vue Router? Yes
# ✔ Add Pinia? No
# ✔ Add Vitest? No
# ✔ Add ESLint? Yes

# Navigate and install
cd my-vue-app
npm install
npm run dev

Project Structure

A typical Vue project created with Vite has this structure:

my-vue-app/
├── node_modules/
├── public/
│   └── favicon.ico
├── src/
│   ├── assets/
│   ├── components/
│   │   └── HelloWorld.vue
│   ├── App.vue
│   └── main.js
├── index.html
├── package.json
└── vite.config.js

Understanding the Files

File Purpose
index.html Entry HTML file
src/main.js Application entry point
src/App.vue Root component
src/components/ Reusable Vue components
package.json Project dependencies and scripts
vite.config.js Vite build configuration

The main.js File

This is where your Vue application is created and mounted:

import { createApp } from 'vue'
import App from './App.vue'

createApp(App).mount('#app')

Single File Components (.vue)

Vue components are typically written in .vue files with three sections: 

<template>
  <div class="hello">
    <h1>{{ msg }}</h1>
  </div>
</template>

<script>
export default {
  name: 'HelloWorld',
  data() {
    return {
      msg: 'Welcome to Vue!'
    }
  }
}
</script>

<style scoped>
.hello {
  color: #42b983;
}
</style>
✅
Best Practice

Use the scoped attribute on <style> tags to ensure CSS only applies to the current component, preventing style conflicts.

⚠️
Common Mistake

Make sure Node.js is installed on your system before using npm or create-vue. Download it from nodejs.org.

Lesson 03

Components

Building reusable UI pieces

What are Components?

Components are reusable Vue instances with a name. They accept inputs (props), manage their own state (data), and can emit events to communicate with parent components.

Think of components as custom HTML elements that encapsulate markup, logic, and styling.

Creating a Component

Here's a simple component example:

<!-- ButtonCounter.vue -->
<template>
  <button @click="count++">
    Count is: {{ count }}
  </button>
</template>

<script>
export default {
  name: 'ButtonCounter',
  data() {
    return {
      count: 0
    }
  }
}
</script>

Using Components

To use a component, import it and register it:

<!-- App.vue -->
<template>
  <div>
    <h1>My App</h1>
    <ButtonCounter />
    <ButtonCounter />
  </div>
</template>

<script>
import ButtonCounter from './components/ButtonCounter.vue'

export default {
  components: {
    ButtonCounter
  }
}
</script>

Props (Parent to Child Communication)

Props allow you to pass data from a parent component to a child component: 

<!-- UserCard.vue -->
<template>
  <div class="card">
    <h2>{{ name }}</h2>
    <p>{{ email }}</p>
  </div>
</template>

<script>
export default {
  props: {
    name: {
      type: String,
      required: true
    },
    email: {
      type: String,
      default: 'No email provided'
    }
  }
}
</script>

<!-- Usage in parent -->
<UserCard name="John Doe" email="john@example.com" />

Custom Events (Child to Parent Communication)

Components can emit custom events to communicate with parents:

<!-- Child component -->
<template>
  <button @click="handleClick">Click me</button>
</template>

<script>
export default {
  methods: {
    handleClick() {
      this.$emit('custom-event', 'Hello from child!')
    }
  }
}
</script>

<!-- Parent component -->
<ChildComponent @custom-event="handleCustomEvent" />

Component Lifecycle

Vue components have lifecycle hooks you can tap into:

Hook When It Runs
beforeCreate Before instance is initialized
created After instance is created
beforeMount Before component is mounted to DOM
mounted After component is mounted to DOM
beforeUpdate Before reactive data changes update DOM
updated After DOM is updated
beforeUnmount Before component is destroyed
unmounted After component is destroyed 
export default {
  mounted() {
    console.log('Component is now mounted!')
  },
  unmounted() {
    console.log('Component is being destroyed')
  }
}
✅
Best Practice

Use mounted() for API calls and DOM manipulation. Use created() for data initialization that doesn't require the DOM.
